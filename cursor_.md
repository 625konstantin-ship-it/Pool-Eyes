## Как устроен сайт (по шагам)

### Шаг 1 — `index.html` — «скелет» страницы

HTML описывает, **что** есть на странице:

- **Настройки бассейна** — выбор объёма (10 000 / 25 000 / 50 000 / 75 000 л или свой).
- **Форма измерения** — поля для pH, хлора и температуры.
- **Рекомендации** — блок, куда подставляются советы.
- **Графики** — три области для Chart.js.
- **Таблица истории** — все прошлые измерения.

В конце подключаются `styles.css`, Chart.js (из интернета) и `app.js`.

### Шаг 2 — `styles.css` — «одежда» страницы

CSS задаёт **как** всё выглядит:

- голубой фон в стиле воды;
- белые карточки с тенью;
- зелёные / жёлтые / красные подсказки по статусу;
- адаптив для телефона.

### Шаг 3 — `app.js` — «мозг» приложения

JavaScript делает сайт живым:

1. **Сохранение** — данные пишутся в `localStorage` браузера (отдельный сервер не нужен).
2. **Рекомендации** — по нормам:
   - pH: **7.2–7.6**
   - хлор: **1.0–3.0 мг/л**
   - температура: **24–28 °C** (комфорт)
3. **Графики** — линии по истории измерений.
4. **Таблица** — новые записи сверху, статус: «Норма» / «Внимание» / «Критично».

---

## Как пользоваться

1. Выберите объём бассейна (или «Свой объём...» и введите литры).
2. Введите **pH**, **свободный хлор** и **температуру**.
3. Нажмите **«Сохранить измерение»**.
4. Смотрите **рекомендации**, **графики** и **историю** ниже.

Пример для проверки: pH `7.4`, хлор `1.5`, температура `26` — должны быть зелёные советы «в норме».

Чтобы увидеть графики, добавьте **2–3 измерения** в разные дни (можно с разными значениями).

---

## Важно знать

- Данные хранятся **только в этом браузере** на этом компьютере.
- Очистка данных браузера удалит историю.
- Кнопка **«Очистить историю»** удаляет все записи без восстановления.
- Рекомендации ориентировочные; перед дозировкой химии сверяйтесь с инструкцией на упаковке.

## Что добавилось

### 1. Выбор активного бассейна
Вверху блока «Настройки бассейна» — выпадающий список. Все измерения, графики и рекомендации относятся **только к выбранному** бассейну.

### 2. Добавление нового бассейна
- **Название** — например: «Дача», «Отель», «СПА»
- **Объём** — как раньше (готовые варианты или свой)
- Кнопка **«+ Добавить бассейн»**

После добавления новый бассейн сразу становится активным.

### 3. Удаление бассейна
Кнопка **«Удалить»** рядом со списком. Последний бассейн удалить нельзя. При удалении стираются и все его измерения.

### 4. Объём для каждого бассейна
Внизу блока настроек — объём **выбранного** бассейна. Меняете объём — он сохраняется для этого бассейна отдельно.

3. Добавьте второй: название «Дача», объём 10 000 л → **«+ Добавить бассейн»**.
4. Переключайтесь в списке **«Активный бассейн»** — история и графики меняются.
5. Вводите измерения для каждого бассейна по отдельности.

## Что исправлено

### Сохранённые бассейны

### Свой объём
- При выборе **«Свой объём...»** появляется **синее поле** для ввода литров.
- Работает и при добавлении нового бассейна, и при изменении существующего.
- Добавлена кнопка **«Сохранить объём»** — нажмите её после ввода своего объёма.

### Надёжное сохранение
Создан файл **`запуск.command`** — лучший способ открыть сайт:

1. Дважды кликните **`запуск.command`** в папке `проэкт`
2. Не закрывайте появившееся чёрное окно
3. В браузере откройте: **http://localhost:8080**
4. Обновите страницу (**F5** или **Cmd+R**)
