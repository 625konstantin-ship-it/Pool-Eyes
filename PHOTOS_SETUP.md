# Фото бассейнов — настройка Supabase

Фото хранятся в **Supabase Storage** (облако). К каждому бассейну можно добавлять снимки с телефона или компьютера.

---

## Шаг 1. SQL в Supabase (один раз)

1. Откройте [supabase.com](https://supabase.com) → ваш проект  
2. **SQL Editor** → **New query**  
3. Скопируйте **весь** файл `supabase/photos.sql` и нажмите **Run**  
4. Должно быть **Success**

Это создаст:
- таблицу `pool_photos` (подписи и ссылки на файлы)
- bucket `pool-photos` (сами файлы)
- правила доступа (каждый видит только свои фото)

> Если политика Storage уже существует и будет ошибка «policy already exists» — удалите старые политики `pool_photos_storage_*` в **Storage → Policies** и запустите SQL снова.

---

## Шаг 2. Проверьте Storage

1. **Storage** → должен появиться bucket **`pool-photos`**  
2. **Public bucket** — **выключен** (это нормально, фото приватные)  
3. **Policies** — 3 политики: select, insert, delete  

---

Обновите на сайте:

| Файл |

| `index.html` |

Обновите страницу с очисткой кэша (**Ctrl+Shift+R**).

---

## Шаг 4. Проверка на сайте

1. Войдите в аккаунт  
2. Выберите бассейн  
3. Блок **«Фото бассейна»** → **Добавить фото**  
4. Выберите снимок с телефона или компьютера  
5. Фото должно появиться в галерее  

Откройте сайт на другом устройстве — те же фото должны быть видны.

---

## Ограничения

- Форматы: **JPG, PNG, WebP**
- Размер: до **5 МБ** на файл
- На бесплатном Supabase: **~1 ГБ** хранилища на весь проект

---

## Если не работает

| Ошибка | Что делать |
|--------|------------|
| `relation "pool_photos" does not exist` | Запустите `photos.sql` в SQL Editor |
| `Bucket not found` | Проверьте bucket `pool-photos` в Storage |
| `new row violates row-level security` | Перезапустите `photos.sql` (политики) |
| `Payload too large` | Фото больше 5 МБ — сожмите или выберите меньшее |

---

## Где лежат файлы

Путь в Storage: `{ваш_user_id}/{id_бассейна}/{id_фото}.jpg`

Удаление бассейна удаляет и все его фото.
